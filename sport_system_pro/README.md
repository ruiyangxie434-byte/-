# 校园运动打卡与健康数据分析系统（豪华完善版）

这是一个使用 **Python 标准库 tkinter + csv/json** 开发的校园运动健康管理系统，适合作为 Python 课程实训、大作业、答辩展示项目。

本版本重点解决了上一版“字体偏小、看起来费劲”的问题：整体界面字体、按钮、表格行高、标签页字号都已放大，并改成多标签页结构，功能层次更清晰。

## 一、新版核心亮点

### 1. 界面优化

- 全局字体放大到 12 号左右，标题、卡片数字、表格行高进一步加大。
- 新增多标签页：数据首页、学生端打卡、健康档案、健康自测、智能分析、教师端、管理员端、社交激励/体测。
- 数据卡片、趋势图、统计摘要、表格明细分区展示，答辩观感更完整。

### 2. 学生端功能

- 学号、姓名、班级、角色基础信息。
- 个人健康档案：身高体重、体脂率、血压、肺活量、既往伤病、过敏史、体检备注。
- 目标自定义：周运动时长、每日步数目标、减脂/增肌/体测达标。
- 运动打卡：支持 GPS 实景打卡、手动录入、设备同步、补卡申请。
- 打卡分类：跑步、球类、健身、早操、课外体育、体育课实训、体测专项。
- 支持时长、距离、步数、心率、热量估算、地点、备注/凭证说明。

### 3. 健康自测功能

- 记录晨起体重、睡眠时长、三餐饮食、当日心率、疲劳程度、伤病记录。
- 提供女生生理期备注字段，可用于后续智能避开高强度运动推荐。
- 保存后可在表格中查看和删除。

### 4. 智能分析模块

- BMI 自动计算与状态判断。
- 风险预警：久坐/缺运动、运动过量、单次运动过长、睡眠不足、疲劳、体重骤变。
- 个性化运动方案：减脂、有氧；增肌、力量训练；体测达标、专项训练。
- 饮食建议：根据运动消耗和目标给出三餐建议。
- 体测预判：结合历史体测和运动数据预测综合分与评级。
- 可视化：最近 7 天运动时长柱状图、体重趋势折线图。

### 5. 教师端功能

- 班级运动数据统计：有效次数、总时长、总热量、待审批数量。
- 任务发布：可以模拟布置课后运动作业、设置地点和有效期。
- 补卡/异常审核：通过选中补卡，或驳回并标记异常。
- 班级报表导出：生成 txt 报表。

### 6. 管理员端功能

- 校园打卡点位设置：操场、篮球场、田径场、体育馆等。
- 打卡规则配置：最短有效时长、积分计分规则。
- 黑名单管理：频繁代打卡学生可限制打卡权限。
- 全校大数据总览：总打卡数、有效记录、待审批、累计时长、热量、步数、体测达标率。
- 预留扩展接口说明：教务系统、GPS 围栏、手环 APP、医务室联动。

### 7. 社交激励与体测

- 运动排行榜：按总时长、热量、积分排名。
- 积分商城说明：饮用水、运动小礼品、体育课加分申请。
- 体测专项管理：录入 800/1000 米、立定跳远、肺活量、坐位体前屈、总分。
- 自动评级：优秀、良好、及格、不及格。
- 不及格自动生成补强建议。

## 二、运行环境

- Python 3.10+
- PyCharm Community / Professional
- 不需要安装第三方库，只使用 Python 标准库

## 三、运行方式

1. 解压项目压缩包。
2. 使用 PyCharm 打开项目文件夹。
3. 找到并运行 `main.py`。
4. 首次运行会自动读取/创建 `data` 目录中的 CSV/JSON 数据。

## 四、文件说明

| 文件/文件夹 | 说明 |
| --- | --- |
| `main.py` | 主程序入口，负责 Tkinter 多标签页界面和交互逻辑 |
| `models.py` | 数据模型，定义打卡记录、健康档案、健康自测、体测记录 |
| `storage.py` | CSV/JSON 数据读写、备份、导出模块 |
| `analytics.py` | 热量估算、BMI、风险预警、排行榜、体测评级、报告生成 |
| `data/sport_records.csv` | 运动打卡数据 |
| `data/profile.json` | 个人健康档案与目标设置 |
| `data/health_logs.csv` | 日常健康自测数据 |
| `data/physical_tests.csv` | 体测专项成绩数据 |
| `reports/` | 导出的健康周报、班级报表 |
| `backups/` | 数据备份文件 |

## 五、答辩时可以这样介绍

1. 本系统采用模块化结构，分为模型层、存储层、分析层、界面层。
2. 学生端完成运动打卡、健康档案、健康自测和目标设置。
3. 系统通过 MET 公式估算运动热量，并根据 BMI、运动频率、睡眠、疲劳程度生成风险预警。
4. 教师端可以查看班级打卡率、审核补卡、导出班级数据。
5. 管理员端可以配置校园打卡点位、打卡规则、黑名单和全校数据总览。
6. 系统预留教务系统、GPS 围栏、运动手环、小程序/Web 后台扩展接口。

## 六、核心计算公式

```text
运动消耗 kcal = MET × 体重 kg × 运动时长 h
BMI = 体重 kg / 身高 m²
```

不同运动类型对应不同 MET 参考值，例如：足球 7.5、跑步 8.0、跳绳 9.0、力量训练 6.0。
