# v3.0 改进清单

## 架构与数据

- 主程序从 CSV/JSON 切换为 SQLite DAO，旧文件仅保留用于首次迁移。
- 新增 `repositories.py`，按登录角色控制查询范围，并阻止学生写入他人数据。
- 管理员点位、黑名单、最短有效时长、积分规则和教师任务改为持久化设置。
- 数据库 Schema 升级到 v2，新增 `settings` 表和自动迁移。

## 登录与安全

- 新增独立登录窗口和学生/教师/管理员三类会话。
- 移除可随意切换角色的下拉框；标签页按真实登录角色显示。
- 学号、姓名和角色等身份字段只读，关键写操作增加后端权限校验。
- 密码改为随机盐 PBKDF2-HMAC-SHA256（260,000 次迭代），支持旧哈希兼容验证。
- 删除健康数据前增加二次确认；数据库备份仅管理员可执行。

## 界面与体验

- 所有页面接入垂直滚动容器，修复 1280/1400 宽窗口下的纵向裁切。
- 教师端双大表、社交与体测区域改为纵向排列，降低横向拥挤。
- 学生只显示 6 个相关标签，教师和管理员各显示 4 个标签。
- 接入 Matplotlib 图表组件，并增加无数据空状态。
- 运动数据导出改为格式化 Excel，周报和 SQLite 备份接入现有独立模块。
- 标题、版本、账号状态和“安全退出”统一到主窗口头部。

## 逻辑、校验与质量

- 增加身高、体重、体脂、心率、时长、体测年份和总分范围校验。
- “偏好运动”只统计审核通过的记录。
- 体测预判只使用近 30 天运动量，不再累计全部历史数据。
- 将界面中的“智能分析”说明明确为可解释的健康规则引擎。
- 测试由 44 项增加到 48 项，新增随机盐、默认账号登录、设置持久化和学生数据隔离测试。
- README 与真实实现重新对齐，并明确 GPS/手环为模拟功能。
