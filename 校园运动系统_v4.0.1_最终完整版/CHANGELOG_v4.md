# v4.0 修改清单

## v4.0.1 热修复

- 登录窗口高度适配 Windows 显示缩放，避免登录按钮被挤出窗口。
- 首次登录强制改密窗口不再绑定到隐藏主窗口，并主动前置显示。

## P0

- 计分规则改为 6 字段 JSON，并由排行榜实时读取。
- 黑名单校验下沉到 `SportRepository.save_record`。
- 学生打卡体重只读，仓储层使用健康档案体重重算热量。
- `users` 新增初始密码和账号状态；首次登录强制改密。
- 删除、审核改为按记录 ID 的增量操作。
- 管理员新增完整用户管理区。

## P1

- 新增 `services.py` 业务服务层。
- 拆分运动、健康、体测的按需刷新流程，并缓存图表签名。
- SQLite 分页查询，默认 40 条/页。
- 运动与健康数据新增联合唯一索引。
- 恢复 SQLite 原生同线程保护。

## P2

- 数字输入限制与失焦内联校验。
- 备份、导出和批量导入异步执行。
- 四项体测自动估分和评级。
- 新增学生名单、运动记录、体测成绩 CSV/XLSX 批量导入。

## UI

- 接入 ttkbootstrap flatly。
- 顶部科技蓝到青绿渐变标题栏。
- 顶部 Notebook 改为左侧图标导航，按角色生成菜单。
- 首页指标卡改为六色渐变卡片。
- 表格使用蓝色表头、斑马纹、悬停高亮和状态色。
- 输入框焦点主色边框，按钮区分主/成功/危险/描边层级。

## 安全与长期演进

- 新增本地图形验证码用于切换账号。
- 新增审计表并记录改密、打卡、删除、审核、用户和计分规则操作。
- SQLCipher、真实短信/邮件验证码和备份恢复保留为外部依赖扩展项。
