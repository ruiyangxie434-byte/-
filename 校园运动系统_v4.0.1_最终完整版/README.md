# 校园运动健康数据中台 v4.0.1

基于 Python、Tkinter/ttkbootstrap、SQLite 和 Matplotlib 的校园运动健康管理桌面系统。v4 重点完成业务正确性、安全闭环、大数据分页和企业级界面升级。

## 安装与运行

```powershell
python -m pip install -r requirements.txt
python main.py
```

第一次使用默认账号登录时，系统会强制修改初始密码，不修改无法进入主界面。

| 角色 | 初始账号 | 初始密码 |
| --- | --- | --- |
| 学生 | `20250101` | `123456` |
| 体育老师 | `T001` | `teacher123` |
| 管理员 | `A001` | `admin888` |

## v4 核心能力

- `ttkbootstrap flatly` 主题、科技蓝渐变标题栏、左侧角色导航、渐变指标卡、现代按钮/输入框/表格。
- SQLite Schema v4：初始密码标记、账号启停、业务唯一索引、审计日志和结构化设置。
- 强制修改初始密码；主界面支持修改密码。
- 切换账号前使用本地图形验证码做人机验证。它不发送短信/邮件；接入真实验证码需要短信或邮件服务配置。
- 管理员用户管理：新增用户、修改姓名/角色/班级、重置临时密码、启用/禁用账号。
- 动态积分：基础分、分钟换算、单次奖励和满额奖励均由结构化 JSON 配置驱动。
- 黑名单在仓储层统一拦截；学生打卡热量强制使用健康档案体重重新计算。
- 运动删除和教师审核使用记录 ID 执行增量 `DELETE/UPDATE`，不再全表重写。
- 运动记录分页查询，默认每页 40 条；筛选条件在 SQLite 查询层生效。
- 运动和健康记录具备业务唯一约束，拒绝同日重复刷数据。
- 备份、Excel 导出和批量导入在后台线程执行，降低界面卡顿。
- 管理员支持 CSV/XLSX 批量导入学生名单、运动记录和体测成绩，并返回逐行错误明细。
- 体测总分根据已采集四项自动估算并评级；界面明确说明并非完整国家标准成绩单。
- 关键操作写入 `audit_logs`，为后续管理员审计页面提供数据基础。

## 默认计分规则

```json
{
  "base_points": 1,
  "minutes_per_point": 10,
  "session_bonus_threshold": 30,
  "session_bonus_points": 5,
  "full_bonus_threshold": 150,
  "full_bonus_points": 20
}
```

排行榜实时从数据库读取规则。管理员保存规则后会立即重新计算积分。

## 批量导入表头

- 学生名单：`账号/学号、姓名、角色、班级、初始密码`
- 运动记录：`学号、姓名、班级、日期、运动类型、打卡分类、打卡方式、地点、时长(分钟)、距离(km)、步数、心率、体重(kg)、审核状态、备注`
- 体测成绩：`学号、姓名、性别、年份、跑步单项分、立定跳远(cm)、肺活量、坐位体前屈(cm)、总分`

支持 `.csv` 和 `.xlsx`。

## 模块结构

```text
main.py          ttkbootstrap GUI、页面与事件绑定
services.py      登录、打卡、审核、用户和配置业务规则
repositories.py  数据范围、黑名单、档案体重等仓储约束
database.py      SQLite Schema、DAO、分页、认证、审计、迁移与备份
auth_ui.py       登录、强制改密、修改密码和图形验证码
importer.py      CSV/XLSX 批量导入与错误明细
analytics.py     统计、动态积分、健康规则和体测估分
charts.py        Matplotlib 图表
export.py        Excel/CSV/文本报告导出
models.py        数据模型和范围校验
storage.py       旧 CSV/JSON 迁移兼容
tests/           pytest 测试
```

## 仍属于扩展项

- SQLCipher 数据库文件加密需要额外原生依赖，当前未伪装为已实现。
- 真实短信/邮件验证码需要用户联系方式、服务商账号和网络 API。
- GPS、运动手环仍是演示入口，没有连接真实第三方平台。
- 完整体测国标总分还需要采集 BMI、50 米、引体向上/仰卧起坐等缺失项目。
